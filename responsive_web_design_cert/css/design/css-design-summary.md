# UI Design Fundamentals: Complete Guide

## 1. Common Design Terms
Design terminology bridges the gap between design and development.

* **Typography**: Serif vs Sans Serif; hierarchy communicates importance.
* **Spacing**: Kerning (letters), tracking (groups), leading (line-height).
* **Whitespace**: Improves clarity and reduces clutter.
* **Design Deliverables**:
  * Wireframes (structure)
  * Mockups (visual detail)
  * Prototypes (interaction)

---

## 2. Contrast (Accessibility & Readability)
Contrast ensures content is visible and accessible.

* **WCAG Standards**:
  * 4.5:1 (normal text)
  * 3:1 (large text)
* **Best Practices**:
  * Use overlays, blur, or shadows for text on images
  * Prioritize **value (light/dark)** over color differences
* **Use Case**:
  * High contrast → primary actions (CTAs)
  * Low contrast → secondary content

---

## 3. Visual Hierarchy
Hierarchy guides user attention and interaction.

* **Tools**:
  * Scale
  * Color
  * Typography
  * Whitespace
* **Patterns**:
  * F-pattern (content-heavy)
  * Z-pattern (simple layouts)
* **Technique**: Squint test to validate emphasis

---

## 4. Scale
Scale defines relationships between elements.

* Relative, not absolute
* Larger elements carry more visual weight
* Use **type scales** (e.g., 1.25 ratio) for consistency
* Creates rhythm, emphasis, and balance

---

## 5. Alignment
Alignment creates structure and cohesion.

* Types:
  * Edge alignment
  * Center alignment
  * Optical alignment
* Use grid systems for consistency
* Avoid mixing alignments unnecessarily

---

## 6. Whitespace
Whitespace enhances usability and aesthetics.

* **Types**:
  * Macro (layout spacing)
  * Micro (text spacing)
* **Functions**:
  * Improves readability
  * Creates focus
  * Groups related elements
* More whitespace → elegance; less → density

---

## 7. Images
Images should enhance—not distract from—content.

* **Best Practices**:
  * Use high-quality, relevant images
  * Optimize formats (WebP, JPEG, SVG)
  * Maintain consistent style
* **Accessibility**:
  * Provide alt text
* **Performance**:
  * Compress and lazy-load images

---

## 8. Progressive Enhancement
Build experiences from basic to advanced.

* **Layered Approach**:
  1. HTML (content)
  2. CSS (presentation)
  3. JavaScript (interaction)
* **Benefits**:
  * Accessibility
  * Performance
  * Resilience
* Preferred over graceful degradation
